## 수집 품질 기준

- 한 이슈당 공식/스토어/미디어 출처를 최소 2개 이상 대조합니다. 단, 공식 릴리즈만 있는 경우에는 공식 릴리즈 문구와 화면 캡처 근거를 더 자세히 남깁니다.
- 기능명만 적지 않고, 사용자가 어느 화면에서 어떤 행동을 덜 하게 되는지까지 적습니다.
- 수치가 있으면 기간, 비교 기준, 출처를 함께 적습니다. 수치가 없으면 “미공개”라고 발행용 문장에 쓰지 않고 수집 메모에만 남깁니다.
- 이미지가 있으면 이미지 URL, 이미지가 보여주는 UI 변화, 캡처/확인 날짜를 함께 남깁니다.
- 매거진 상세에 쓸 수 있도록 장면, 문제의식, 인용형 리드, 마지막 관점까지 초안 재료를 모읍니다.
- DEV 후보는 단순 개발자 뉴스가 아니라 UIUX 팀의 화면 구현, 디자인 시스템, 접근성 QA, 브라우저 호환성 판단에 도움이 되는 정보만 채택합니다.
- DEV 후보는 금주 기준 최신 기술을 우선하며, AI는 최우선으로 검토합니다.
- DEV 수집본에는 접근성, CSS, JavaScript, HTML, iOS, Chrome, Firefox 속성/API 업데이트 사례를 반드시 확인하고, 없는 경우 `제외/주의 사유`에 이번 주 확인 결과를 남깁니다.
