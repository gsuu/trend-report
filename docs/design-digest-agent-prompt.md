# Design Digest Agent Prompt

이 문서는 `npm run fetch:design`으로 모은 `runs/YYYY-MM-DD/design-articles.json`을 읽고, 웹디자이너와 UIUX 디자이너가 저장해둘 만한 디자인 후보를 고르는 기준입니다.

수집 스크립트는 후보만 모읍니다. LLM API를 호출하지 않고, 최종 채택/보류/제외 판단과 원문 확인, 요약 작성은 작업 AI가 이 문서를 기준으로 수행합니다.

## 목적

DESIGN 후보의 중심은 "디자이너가 다음 시안에 참고할 수 있는가"입니다.

좋은 후보는 세 가지 중 하나에 해당해야 합니다.

1. 웹디자이너가 보고 혹할 만한 감각 있는 레퍼런스
2. 지금 웹/브랜드/콘텐츠 디자인에서 보이는 최신 표현 트렌드
3. 개발 구현이 아니라 디자이너의 제작, 표현, 협업에 도움이 되는 실무 기술

디자인 시스템과 도구 업데이트도 중요하지만, 그것만으로는 부족합니다. 히어로 섹션, 타이포그래피, 컬러 조합, 이미지 연출, 카드 구성, 룩북형 상세 페이지, 브랜드 캠페인 페이지, 스크롤 인터랙션, 에디토리얼형 커머스처럼 웹디자이너가 바로 저장하고 싶은 장면을 적극적으로 찾습니다.

국내 독자가 바로 참고할 수 있도록 한국 UIUX 사례와 국내 프로모션/이벤트 디자인 사례를 우선 검토합니다. 네이버, 카카오, 토스, 삼성, 무신사, 29CM, 올리브영, SSG 같은 국내 서비스/커머스/브랜드 사례는 화면 구조, 캠페인 콘셉트, 룩북/기획전 구성, 온오프라인 연결 방식이 확인되면 해외 레퍼런스보다 먼저 채택 후보로 봅니다. 해외 자료는 국내 후보만으로 디자인 관점이 부족할 때 보완용으로 사용합니다.

## 입력

- `runs/YYYY-MM-DD/design-articles.json`
- 필요 시 원문 URL
- 필요 시 후보 발견 출처가 가리키는 공식 발표, 제품 문서, 릴리즈 노트, 쇼케이스, 사례 페이지

## 출력

`shortlist-20-30.md` 또는 `magazine-report.md`에 넣기 전에 각 후보마다 아래 판단을 남깁니다. `magazine-report.md`에 올릴 때는 Service 템플릿을 쓰지 않고 Design 전용 템플릿을 씁니다.

- 판정: `채택 / 보류 / 제외`
- 디자인 가치: `web_reference / visual_trend / brand_expression / layout_pattern / motion_interaction / practical_skill / system / method / exclude`
- 직무 태그: `웹디자인`
- 출처 유형: `news / release_note / blog_opinion / reference / guide / research`
- 저장할 이유: 웹디자이너가 왜 저장할 만한지 한 문장
- 실무 적용 포인트: 다음 시안이나 운영 디자인에 어떻게 써먹을 수 있는지 한 문장
- 원문 확인 URL
- 제외 사유: 제외일 때만 작성

## 발행 템플릿

Design 글은 아래 구조를 사용합니다.

각 Design 항목은 메타데이터에 `출처 유형`을 반드시 작성합니다. 실제 웹사이트, 쇼케이스, 브랜드 사례, 포트폴리오처럼 화면을 보는 글은 `reference`, 공식 제품 업데이트는 `release_note`, 디자인 방법론이나 사용성 가이드는 `guide`, 리서치나 리포트는 `research`, 인터뷰나 관점 중심 글은 `blog_opinion`, 일반 기사 성격은 `news`를 사용합니다.

제목은 후보 성격과 원문 맥락을 먼저 보고 정합니다. 화면 레퍼런스, 브랜드 사례, 캠페인 사례는 독자가 볼 포인트를 바로 알 수 있는 관점형 제목을 허용합니다. 반대로 NN/g처럼 가이드·리서치·방법론을 정리한 글은 인사이트 문장으로 바꾸기보다 `출처 + 정리한 주제 + 자료 성격`이 드러나야 합니다. 예: `Nielsen Norman Group이 정리한 웹사이트 AI 챗봇 설계를 위한 10개 가이드라인`. 이런 글에서는 `사이트 AI 챗봇은 먼저 자기 역할을 밝혀야 한다`처럼 본문 인사이트에 가까운 문장을 제목으로 올리지 않습니다.

```markdown
- 출처 유형: reference

##### 디자인 레퍼런스 요약

- 레퍼런스:
- 볼 포인트:
- 기존 방식:
- 표현 방식:
- 확인한 근거:

##### 디자인 인사이트

> 저장할 이유를 압축한 한 문장

###### 왜 참고할 만한가

###### 어디에 적용할 수 있을까

###### 디자인 관점

###### 점검 질문
```

`서비스 변화 요약`, `서비스 맥락`, `변경 전/후`, `사용자는 무엇을 덜 해도 될까` 라벨은 Design 글에 쓰지 않습니다.

## 먼저 보는 질문

후보를 볼 때 아래 질문을 먼저 적용합니다.

- 웹디자이너가 "이거 저장해둬야겠다"고 느낄 장면이 있는가?
- 다음 시안에 적용할 수 있는 레이아웃, 톤, 이미지 연출, 인터랙션, 콘텐츠 구조가 있는가?
- 커머스, 브랜드, 캠페인, 랜딩, 상세 페이지 작업에 바로 참고할 수 있는가?
- 국내 서비스, 커머스, 프로모션 이벤트 작업에 바로 비교하거나 응용할 수 있는가?
- 요즘 디자인 감각이나 표현 방향을 읽는 데 도움이 되는가?
- 단순히 예쁜 결과물이 아니라 어떤 표현 방식, 제작 방식, 선택 기준을 배울 수 있는가?

## 채택 기준

아래 중 하나 이상이 원문에서 구체적으로 확인되면 채택 후보입니다.

1. 웹 레퍼런스
   랜딩 페이지, 브랜드 사이트, 캠페인 페이지, 포트폴리오, 커머스 상세, 룩북, 에디토리얼 페이지처럼 화면 단위로 참고할 만한 사례가 있다.

1-1. 국내 UIUX/프로모션 사례
   한국 서비스의 리뉴얼, 앱/웹 UX 변화, 커머스 기획전, 브랜드위크, 멤버십 이벤트, 팝업/오프라인 연계 캠페인처럼 국내 웹디자이너가 다음 운영 디자인에 바로 참고할 만한 사례가 있다.

2. Look & Feel
   타이포그래피, 컬러, 그리드, 여백, 이미지 톤, 일러스트, 3D, 그래픽, 사진 연출처럼 시각적 분위기를 참고할 만하다.

3. 레이아웃 패턴
   히어로 섹션, 카드/리스트, 제품 모듈, 비교 영역, 스토리텔링 섹션, CTA, 하단 구성처럼 웹 화면에 바로 응용할 수 있는 구성이 있다.

4. 브랜드 표현
   브랜드 아이덴티티, 캠페인 비주얼, 콘텐츠 톤, 마이크로카피, 세계관, 아트 디렉션처럼 브랜드 경험을 만드는 방식이 보인다.

5. 모션과 인터랙션
   스크롤 연출, 전환, 마이크로 인터랙션, 3D, 영상/모션 사용 방식처럼 움직임이 경험을 설득하는 사례다.

6. 최신 디자인 트렌드
   AI 생성 비주얼, 대담한 타이포그래피, 에디토리얼형 커머스, 미니멀한 고급감, 소셜/숏폼 감각의 구성, 몰입형 스토리텔링처럼 최근 표현 흐름을 읽을 수 있다.

7. 실무 제작 기술
   Figma, Adobe, Canva, Framer, Webflow 등에서 디자이너의 시안 제작, 이미지 생성/보정, 브랜드 키트, 프로토타입, 프레젠테이션, 콘텐츠 제작 흐름이 달라진다.

8. 디자인 시스템과 UX 방법론
   컴포넌트, 토큰, 라이브러리, 문서화, 사용성 평가, 정보구조, 콘텐츠 전략처럼 실무 판단이나 운영 기준에 도움 된다.

## 제외 기준

아래 후보는 원칙적으로 제외합니다.

- 결과 이미지는 예쁘지만 화면 구조, 톤, 제작 방식, 적용 포인트를 설명하기 어려운 글
- 브랜드 캠페인 소개지만 웹디자이너가 참고할 화면, 콘텐츠 구조, 비주얼 시스템이 없는 글
- 개발 구현, 브라우저, CSS/JS/API 중심 글. 이런 글은 DEV 후보로 보냅니다.
- 투자, 실적, 조직, 행사 참가 소식처럼 디자인 실무 적용 포인트가 없는 글
- "AI로 더 쉽게 만든다" 정도의 홍보 문구만 있고 실제 입력 방식, 결과물, 작업 흐름이 없는 글
- 쇼케이스 페이지의 썸네일만 있고 원문 사이트나 실제 화면을 확인할 수 없는 글

## 디자인 가치 분류

- `web_reference`: 웹사이트, 랜딩, 커머스, 포트폴리오, 브랜드 사이트 레퍼런스
- `visual_trend`: 최신 시각 표현, 트렌드, 스타일 흐름
- `brand_expression`: 브랜드 아이덴티티, 캠페인, 콘텐츠 톤, 아트 디렉션
- `layout_pattern`: 히어로, 카드, 섹션, CTA, 상세 페이지 등 화면 구성 패턴
- `motion_interaction`: 모션, 스크롤, 전환, 마이크로 인터랙션, 3D
- `practical_skill`: 이미지 제작, 템플릿, 브랜드 키트, 프로토타입 등 디자인 제작 기술
- `system`: 디자인 시스템, 컴포넌트, 토큰, 라이브러리, 문서화
- `method`: 리서치, 사용성, 정보구조, 콘텐츠 전략 같은 방법론
- `exclude`: 디자인 실무 적용 근거가 부족한 후보

## 요약 작성 규칙

채택 후보는 원문을 열고 아래 4개 층위로 요약합니다.

1. 요약
   원문에서 실제로 보여주는 사례나 변화를 1-2문장으로 적습니다.

2. 저장할 이유
   웹디자이너가 이 레퍼런스를 저장할 만한 이유를 구체적으로 적습니다. "예쁘다"가 아니라 어떤 톤, 장면, 구성, 표현 방식 때문인지 씁니다.

3. 디자인 포인트
   레이아웃, 타이포그래피, 컬러, 이미지 연출, 모션, 브랜드 표현, 제작 방식 중 참고할 포인트를 적습니다.

4. 매거진 인사이트
   CTTD 클라이언트 작업에 적용할 질문으로 바꿉니다. 예를 들어 "우리 상품 상세의 히어로 영역은 제품 톤을 충분히 압축해서 보여주는가?"처럼 화면이나 산출물 점검 질문이어야 합니다.

요약, 저장할 이유, 디자인 포인트, 매거진 인사이트는 같은 문장을 반복하지 않습니다.

## 원문 확인 규칙

- 레퍼런스 쇼케이스는 실제 사이트 원문을 열어 확인합니다. 쇼케이스 카드만 보고 발행하지 않습니다.
- 공식 제품 블로그, 릴리즈 노트, 뉴스룸, 디자인 시스템 문서, 제품 업데이트 페이지를 우선합니다.
- 매거진/큐레이션 글에서 발견한 후보는 글 안의 링크, 제품명, 기능명, 발표 날짜를 따라가 공식 원문이나 실제 결과물을 찾습니다.
- 공식 원문이나 실제 사이트를 찾지 못하면 메인 글로 올리지 않고 보류 또는 제외합니다.
- 이미지는 원문 페이지의 대표 이미지, 실제 화면 캡처, 쇼케이스 썸네일 중 디자인 판단에 도움이 되는 것만 씁니다. 분위기만 있는 이미지는 쓰지 않습니다.

## 좋은 후보 예시

- Awwwards, Siteinspire, Land-book에서 발견한 커머스/브랜드/랜딩 페이지 중 레이아웃과 톤이 뚜렷한 사례
- 브랜드 아이덴티티가 웹 화면, 콘텐츠, 캠페인 페이지로 일관되게 확장된 사례
- 에디토리얼형 커머스 상세, 룩북형 상품 페이지, 몰입형 캠페인 페이지
- Framer, Webflow, Figma, Adobe, Canva가 디자이너의 제작 흐름을 줄이거나 표현 폭을 넓힌 업데이트
- NN/g나 UX 매거진이 특정 UI 패턴의 사용성 판단 기준을 제시한 글

## 나쁜 후보 예시

- "새로운 캠페인 공개"인데 웹 화면, 콘텐츠 구조, 비주얼 시스템을 확인할 수 없는 글
- "멋진 사이트 모음"이지만 왜 참고할 만한지 설명할 수 없는 단순 링크
- "AI 기능 출시"지만 입력 방식, 결과물, 디자이너 작업 흐름이 설명되지 않는 글
- CSS, JavaScript, 브라우저 호환성, 접근성 구현 중심 글
- 채용, 투자, 매출, 행사 참가 소식
