# 데이터 수집 전략

이 리포트의 목표는 뉴스를 많이 모으는 것이 아니라, 클라이언트 미팅에서 바로 쓸 수 있는 UIUX·서비스 인사이트를 고르는 것입니다.

## 핵심 원칙

1. 수집은 넓게 합니다.
   국내 모니터링 범위의 모든 서비스를 확인합니다. 패션, 이커머스, 백화점, 뷰티, 도서/콘텐츠 순서는 발행 우선순위이지 수집 제외 기준이 아닙니다.

2. 발행은 좁게 합니다.
   단순 업데이트 10개보다 클라이언트가 바로 질문할 수 있는 강한 이슈 4~7개가 더 좋습니다.

3. UIUX 변화와 서비스 구조를 먼저 봅니다.
   매출, 캠페인, 투자 뉴스도 화면, 탐색, 구매, 재방문, 멤버십, 알림, 오프라인 연계 경험에 영향을 줄 때만 메인 이슈로 올립니다.

4. 수치는 장식이 아니라 근거로 씁니다.
   성장률, 거래액, 전환율, 이용료, 적립률, MAU, 점포 수, 출시일은 기간과 출처가 확인될 때만 씁니다.

5. 이미지는 증거일 때만 씁니다.
   앱스토어 스크린샷, 공식 뉴스룸 이미지, 공식 이벤트 화면처럼 UI나 서비스 구조를 설명하는 이미지를 우선합니다.
   새 후보를 추가한 뒤에는 `scripts/fill_missing_images.py`로 빈 `이미지:`를 먼저 채우고, 자동 탐색으로 못 찾은 경우에만 수집 메모에 사유를 남깁니다.

## 수집 채널

### 1순위 출처

- App Store, Google Play 릴리즈 노트
- 각 서비스 공식 공지, 뉴스룸, 블로그, 이벤트 페이지
- 공식 보도자료
- 공식 앱스토어/플레이스토어 스크린샷

### 2순위 출처

- 신뢰 가능한 경제지, 유통지, IT/스타트업 미디어
- 기업 인터뷰, 컨퍼런스 발표, 투자/실적 관련 공개 자료
- 플랫폼 운영사가 인용된 기사

### 3순위 출처

- 커뮤니티, SNS, 블로그, 채용공고
- 이 출처는 단서로만 사용합니다. 메인 리포트에 쓰려면 1순위 또는 2순위 출처로 재확인합니다.

## 매주 반드시 확인할 축

1. 홈·탐색
   홈 개편, 카테고리 분리, 개인화 피드, 브랜드관, 큐레이션, 검색 결과 UI

2. 비교·선택
   AI 추천, 유사 상품, 듀프, 리뷰 요약, 성분/취향/사이즈 기반 비교, 필터 개선

3. 구매·전환
   장바구니, 결제, 쿠폰, 포인트, 멤버십, 페이백, 구독, 무료배송 조건

4. 재방문·관계
   알림, 찜, 재입고, 출석, 리워드, 멤버십 등급, 앱 전용 혜택

5. 온·오프라인 연결
   매장 재고, 픽업, 선물하기, 백화점/VIP 연계, 팝업, 오프라인 1호점

6. 시장·전략
   해외 진출, 현지 법인, 카테고리 확장, 제휴, 투자, 조직 개편, 거래액/매출 성장

## 채택 점수

각 후보는 10점 기준으로 판단합니다.

1. UIUX·서비스 변화: 0~3점
   화면, 기능, 정책, 혜택 구조 중 실제 변화가 명확할수록 높게 줍니다.

2. 클라이언트 미팅 활용도: 0~3점
   “우리 서비스에서는 무엇을 점검해야 하나?”라는 질문으로 바로 이어지면 높게 줍니다.

3. 근거 신뢰도: 0~2점
   공식 출처와 이미지, 출시일, 원문 URL이 있으면 높게 줍니다.

4. 수치·팩트: 0~1점
   기간이 있는 정량 수치가 있으면 줍니다.

5. 카테고리 우선순위: 0~1점
   패션, 이커머스, 백화점 이슈는 우선 배치합니다. 단, 뷰티/도서라도 UIUX 변화가 강하면 메인 이슈로 올립니다.

8점 이상은 메인 후보, 6~7점은 보류 후보, 5점 이하는 제외 후보로 둡니다.

## 좋은 이슈의 형태

좋은 이슈는 아래 문장에 답할 수 있어야 합니다.

- 무엇이 바뀌었나?
- 사용자는 이전보다 무엇을 덜 하거나 새로 하게 되었나?
- 이 변화가 탐색, 비교, 구매, 재방문 중 어디에 영향을 주나?
- 공개된 수치가 있나?
- 클라이언트 미팅에서 어떤 질문으로 바꿀 수 있나?

## 깊이 수집 프로토콜

채택 후보는 단순 기사 요약이 아니라 아래 5개 층위로 수집합니다. 이 중 3개 이하만 채워진 이슈는 매거진 상세로 쓰기 어렵습니다.

1. 팩트 레이어
   출시일, 버전, 기능명, 정책명, 가격, 적립률, 수치, 출처 URL을 원문 기준으로 남깁니다.

2. 화면·플로우 레이어
   사용자가 어느 화면에서 진입하는지, 어떤 문구와 버튼을 보는지, 이전보다 몇 단계가 줄었는지 적습니다.

3. 사용자 행동 레이어
   사용자가 덜 하게 되는 일, 새로 하게 되는 일, 헷갈릴 수 있는 지점, 재방문 트리거를 적습니다.

4. 비즈니스 맥락 레이어
   멤버십, 거래액, 카테고리 확장, 해외 진출, 재구매, CRM, 오프라인 연계와 어떻게 연결되는지 적습니다.

5. 매거진 서사 레이어
   매거진 인사이트에 쓸 한 문장 리드, 문제의식, 장면, 클라이언트 질문, 마무리 관점을 따로 남깁니다.

## 출처 대조 규칙

- 공식 릴리즈 노트만 있는 이슈는 화면/문구/버전 정보를 자세히 남깁니다.
- 미디어 기사만 있는 이슈는 공식 출처나 앱스토어/플레이스토어에서 재확인합니다.
- 수치가 있는 이슈는 기간, 비교 기준, 원출처를 반드시 남깁니다.
- 이미지가 있는 이슈는 이미지가 어떤 UI 변화를 증명하는지 설명합니다.
- 이미지가 비어 있는 이슈는 발행 전 `scripts/fill_missing_images.py`로 공식 스크린샷이나 대표 이미지를 찾습니다. 못 찾으면 빈칸으로 발행하지 말고 `이미지 확보 메모`에 사유를 남깁니다.
- 출처 간 표현이 다르면 발행문에는 가장 보수적인 표현을 씁니다.

## 예시 판단

무신사 멀티 스토어 홈은 패션 앱의 홈 구조와 카테고리 탐색 방식이 바뀐 사례라서 메인 이슈입니다.

화해 듀프템 찾기는 성분 데이터와 AI 추천이 상품 비교 행동을 줄이는 사례라서 메인 이슈입니다.

G마켓 꼭 멤버십은 단순 프로모션이 아니라 손해 보지 않는다는 가입 설득 UX와 적립 구조가 결합된 신규 서비스 사례라서 메인 이슈입니다.

올리브영 일본 진출은 UI 화면 변경은 아니지만, 글로벌몰 성장, 현지 법인, 오프라인 매장 검토처럼 국내 뷰티 커머스의 옴니채널·크로스보더 전략을 읽을 수 있어 시장·전략 후보로 수집합니다.

단순 “버그 수정 및 안정화”는 결제, 검색, 로그인처럼 핵심 여정 장애를 해결한 구체 내용이 없으면 제외합니다.

## 발행 구조

발행 리포트는 카테고리별로 묶되, 각 카드의 본문은 아래 순서를 유지합니다.

1. 서비스 변화 요약
2. 매거진 인사이트
3. 출처

`서비스 변화 요약`에는 업데이트, 서비스 맥락, 변경 전/후, 실제 수치만 묶습니다. 수치가 없다는 문장은 발행 뉴스레터나 사이트 카드에 쓰지 않습니다.

`매거진 인사이트`는 매거진 상세페이지 전용입니다. 뉴스레터에는 노출하지 않고, 사이트 아티클에서만 보여줍니다. 이 섹션에는 수집된 팩트를 반복하지 말고 아래 내용을 하나의 글 흐름으로 더합니다.

- 배경: 왜 지금 이 업데이트가 나왔는지, 시장/서비스 맥락
- 사용자 여정 해석: 탐색, 비교, 구매, 재방문 중 어떤 행동을 줄이거나 늘리는지
- UIUX 전문가 관점: 화면 위계, 정보구조, CTA, 상태값, 피드백, 추천 근거, 재방문 동선 중 무엇을 설계 관점에서 봐야 하는지
- 클라이언트 적용 포인트: 웹에이전시 미팅에서 던질 수 있는 질문과 적용 아이디어

문체와 구성은 [매거진 상세 글쓰기 가이드](editorial-style-guide.md)를 따릅니다.

## 최종 검수 체크리스트

- 국내 서비스 중심으로 수집했는가?
- 모든 모니터링 서비스를 확인했는가?
- 패션 → 이커머스 → 백화점 → 뷰티 → 도서/콘텐츠 순으로 우선 배치했는가?
- 화면, 기능, 정책, 혜택 구조 중 실제 변화가 있는가?
- 변경 전/후가 사용자 행동 기준으로 설명되는가?
- 수치에는 기간과 출처가 있는가?
- 이미지는 UI나 서비스 구조를 설명하는가?
- 매거진 인사이트가 기사 요약이 아니라 클라이언트 질문으로 이어지는가?
- 매거진 인사이트는 요약 반복이 아니라 배경, 여정 해석, 적용 포인트로 확장됐는가?
- `UIUX 전문가 관점`이 기능 설명이 아니라 디자이너의 화면/플로우 판단으로 작성됐는가?
- 단순 홍보성 문장을 제거했는가?
