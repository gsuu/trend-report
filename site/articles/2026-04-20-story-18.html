2026-04-20 | JAVASCRIPT

기술 변화 요약

  • 업데이트TypeScript 6.0에서 target과 lib에 es2025 옵션이 추가되고, Temporal 타입 지원이 포함됐습니다.
  • 기술 맥락JavaScript 표준 API가 늘어나면서 타입 시스템과 런타임 타깃 설정을 더 명시적으로 맞춰야 하는 상황입니다.
  • 변경 전RegExp.escape, Promise.try, Iterator/Set 계열 메서드 같은 타입은 esnext에 남아 있어 안정 타깃과 실험 타깃의 경계가 흐렸습니다.
  • 변경 후ES2025 내장 API 타입이 별도 타깃으로 정리되고, Temporal은 esnext 또는 granular lib를 통해 타입을 사용할 수 있습니다.
  • 확인할 영향주요 변화 es2025 target/lib, Temporal 타입, DOM iterable 타입 정리 / 확인 한계 Temporal 런타임 지원은 브라우저/서버 환경별 확인이 필요합니다.

인사이트

TypeScript 업그레이드는 문법 문제가 아니라, 팀의 런타임 기준을 명시하는 작업이 되고 있습니다.

왜 지금 이 업데이트인가

TypeScript 6.0의 핵심은 새 문법보다 표준 API 타입의 정리입니다. ES2025 타깃이 생기면서 팀은 "우리가 어떤 JavaScript 표준을 빌드 기준으로 삼는가"를 더 분명하게 설정할 수 있습니다.

Temporal 타입 지원도 중요합니다. 날짜와 시간 처리는 프론트엔드에서 오래된 pain point였고, 표준 API가 타입 레벨에서 보이기 시작하면 라이브러리 의존성, polyfill, 런타임 지원 정책을 다시 점검해야 합니다.

프론트엔드 개발 전문가 관점

프론트엔드 팀은 tsconfig를 프로젝트 초기 설정 파일로만 방치하면 안 됩니다. target/lib 설정은 번들 출력, 타입 체크, 브라우저 호환성, polyfill 전략에 직접 연결됩니다.

특히 디자인 시스템이나 공통 패키지를 운영하는 팀은 ES2025 API를 타입으로 허용할지, 실제 런타임 지원이 확인된 환경에서만 쓸지 기준을 문서화해야 합니다.

실무에 어떻게 적용할 수 있을까

  • UI 컴포넌트 props 타입과 HTML attribute 타입을 정리해 Button, Link, Image 같은 공통 컴포넌트 사용 실수를 줄입니다.
  • target/lib 변경 후 최신 API 사용이 실제 지원 브라우저에서 동작하는지 BrowserStack 또는 실기기 QA 범위에 넣습니다.
  • 날짜/시간 표기가 있는 UI는 Temporal/polyfill 적용 여부와 함께 fallback 문구와 locale 표시가 깨지지 않는지 확인합니다.

같이 보면 좋은 기술

  • tsconfig target/lib: 타입으로 허용하는 표준 API와 실제 빌드 출력 기준을 맞추는 출발점입니다.
  • Browserslist: 서비스가 지원하는 브라우저 범위와 TypeScript lib 설정이 어긋나는지 확인해야 합니다.
  • core-js/polyfill 정책: 타입은 통과하지만 런타임에서 없는 API를 보완할 기준이 필요합니다.
  • Temporal polyfill: Temporal 타입 도입은 실제 브라우저/Node 지원 전까지 polyfill 전략과 함께 검토해야 합니다.
  • package exports 조건: 공통 패키지가 어떤 런타임과 번들러에 어떤 출력을 제공하는지 target 설정과 함께 봐야 합니다.
